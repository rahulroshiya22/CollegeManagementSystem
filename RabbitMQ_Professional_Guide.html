📚 What is RabbitMQ?

RabbitMQ is a message broker - software that helps different applications or services communicate with each other by passing messages. Think of it like a post office: one service (sender) gives a message to RabbitMQ, and RabbitMQ delivers it to another service (receiver).

🎯 Why Do We Need It?

Without RabbitMQ, if your EnrollmentService wants to tell your FeeService that a student enrolled, it would need to:

  • Know the FeeService's exact location (URL)
  • Wait for FeeService to respond
  • Handle errors if FeeService is down

With RabbitMQ: EnrollmentService just sends a message to RabbitMQ and continues working. RabbitMQ handles delivery to FeeService automatically!

🔧 Complete Integration Guide

Let's understand exactly how RabbitMQ ismintegrated into your project, step by step.

1 Install RabbitMQ Client Package

Both services need the RabbitMQ client library to communicate with CloudAMQP.

Terminal
dotnet add package RabbitMQ.Client --version 7.2.0

Why? This package provides the code to connect to RabbitMQ and send/receive messages.

2 Add Connection String

Each service needs to know how to connect to CloudAMQP.

appsettings.json
{
  "CloudAMQP": {
    "ConnectionString": "amqps://username:password@duck.lmq.cloudamqp.com/vhost"
  }
}

Connection String Format:

  • amqps:// - Secure protocol
  • username:password - Your CloudAMQP credentials
  • @duck.lmq.cloudamqp.com - CloudAMQP server
  • /vhost - Virtual host (like a namespace)

3 Create Message Bus (Publisher)

This is the code that sends messages to RabbitMQ.

CMS.Common.Messaging/CloudAMQPBus.cs
public class CloudAMQPBus : IMessageBus
{
    private readonly string _connectionString;

    public CloudAMQPBus(string connectionString)
    {
        _connectionString = connectionString;
    }

    public async Task PublishAsync(string exchange, object message)
    {
        // 1. Create connection factory
        var factory = new ConnectionFactory 
        { 
            Uri = new Uri(_connectionString),
            AutomaticRecoveryEnabled = true  // Auto-reconnect if connection drops
        };
        
        // 2. Open connection and channel
        using var connection = await factory.CreateConnectionAsync();
        using var channel = await connection.CreateChannelAsync();
        
        // 3. Declare exchange (create if doesn't exist)
        await channel.ExchangeDeclareAsync(
            exchange,                    // Name: "student-enrolled"
            ExchangeType.Fanout,        // Type: broadcast to all
            durable: true                // Survive server restart
        );
        
        // 4. Convert message to bytes
        var json = JsonSerializer.Serialize(message);
        var body = Encoding.UTF8.GetBytes(json);
        
        // 5. Publish message
        await channel.BasicPublishAsync(
            exchange,               // Where to send
            "",                       // Routing key (not used with Fanout)
            body                    // Message data
        );
    }
}
💡 Key Points:
  • Exchange: Like a mailbox where messages go
  • Fanout: Sends message to ALL subscribers
  • Durable: Message won't be lost if RabbitMQ restarts
  • AutomaticRecoveryEnabled: Reconnects automatically if connection drops

4 Register in EnrollmentService

Make the message bus available throughout the service.

EnrollmentService/Program.cs
// Register CloudAMQP Message Bus
builder.Services.AddSingleton<IMessageBus>(sp =>

Why Singleton? We want ONE shared connection to RabbitMQ, not a new one for every request.

public async Task CreateEnrollment(Enrollment enrollment)

6 Create Background Subscriber (FeeService)

FeeService runs a background task that continuously listens for messages.

FeeService/Messaging/StudentEnrolledSubscriber.cs
public class StudentEnrolledSubscriber : BackgroundService
{
    private IConnection? _connection;
    private IChannel? _channel;
    private readonly string _connectionString;
    private readonly IServiceProvider _serviceProvider;

    protected override async Task ExecuteAsync(CancellationToken stoppingToken)
    {
        // 1. Connect to CloudAMQP
        var factory = new ConnectionFactory 
        { 
            Uri = new Uri(_connectionString),
            AutomaticRecoveryEnabled = true
        };
        
        _connection = await factory.CreateConnectionAsync();
        _channel = await _connection.CreateChannelAsync();
        
        // 2. Declare exchange (must match publisher)
        await _channel.ExchangeDeclareAsync(
            "student-enrolled",
            ExchangeType.Fanout,
            durable: true
        );
        
        // 3. Create a queue and bind to exchange
        var queueDeclare = await _channel.QueueDeclareAsync();
        await _channel.QueueBindAsync(
            queueDeclare.QueueName,   // Our queue
            "student-enrolled",        // Exchange to listen to
            ""                        // Routing key (not used)
        );
        
        // 4. Start listening for messages
        var consumer = new AsyncEventingBasicConsumer(_channel);
        consumer.ReceivedAsync += OnMessageReceived;
        
        await _channel.BasicConsumeAsync(
            queueDeclare.QueueName,
            autoAck: true,         // Auto-acknowledge receipt
            consumer
        );

        Console.WriteLine("✓ FeeService listening for enrollment events...");
    }

    private async Task OnMessageReceived(object sender, BasicDeliverEventArgs args)
    {
        // 1. Convert bytes back to text
        var body = args.Body.ToArray();
        var json = Encoding.UTF8.GetString(body);
        
        Console.WriteLine($"→ Received from student-enrolled: {json}");
        
        // 2. Deserialize to object
        var enrollmentEvent = JsonSerializer.Deserialize<EnrollmentEvent>(json);
        
        // 3. Process the event (create fee)
        using var scope = _serviceProvider.CreateScope();
        var context = scope.ServiceProvider.GetRequiredService<FeeDbContext>();
        
        var fee = new Fee
        {
            StudentId = enrollmentEvent.StudentId,
            Amount = 5000,  // Default fee
            Description = $"Course Fee for Semester {enrollmentEvent.Semester}",
            Status = "Pending",
            DueDate = DateTime.UtcNow.AddDays(30),
            CreatedAt = DateTime.UtcNow
        };
        
        context.Fees.Add(fee);
        await context.SaveChangesAsync();
        
        Console.WriteLine($"✓ Fee auto-generated for Student {fee.StudentId}: ${fee.Amount}");
    }
}
💡 Background Service: This runs automatically when FeeService starts and keeps running, waiting for messages. It never stops until the service stops.

7 Register Subscriber in FeeService

FeeService/Program.cs

🔄 Complete Message Flow

Let's trace what happens when a student enrolls, step by step:

1

Student Enrolls via API

User sends POST request to: POST https://localhost:7000/api/Enrollment

{
  "studentId": 7,
  "courseId": 2,
  "semester": 1,
  "year": 2024,
  "enrollmentDate": "2024-08-15"
}
2

EnrollmentService Processes

  • Validates the request
  • Saves enrollment to clgmansys3 database
  • Creates enrollment event object
  • Publishes message to RabbitMQ exchange "student-enrolled"
  • Returns success response to user
3

CloudAMQP Receives Message

  • Message arrives at CloudAMQP server
  • Stored in "student-enrolled" exchange
  • Exchange broadcasts to all bound queues
  • FeeService's queue receives the message
4

FeeService Consumes Message

  • Background service detects new message
  • Deserializes JSON to enrollmentEvent object
  • Creates fee record for the student
  • Saves fee to clgmansys4 database
  • Logs success message to console
5

Result

✓ Enrollment saved in EnrollmentService database
✓ Fee automatically created in FeeService database
✓ Services don't know about each other - they only know RabbitMQ!

🎯 This is Event-Driven Architecture!

EnrollmentService doesn't call FeeService directly. It just announces "A student enrolled!" and any service interested in this event can listen and react. This makes the system flexible and easy to extend.

🎓 Key Concepts Explained

Exchange Types

Type How It Works Your Project
Fanout Broadcasts to ALL queues (like a loudspeaker) ✓ Used in your project
Direct Sends to queues with matching routing key Not used
Topic Pattern matching routing (e.g., "student.*") Not used

Important Terms

  • Publisher: Service that sends messages (EnrollmentService)
  • Subscriber/Consumer: Service that receives messages (FeeService)
  • Exchange: Mailbox where publishers send messages
  • Queue: Line where messages wait to be consumed
  • Binding: Connection between exchange and queue
  • Durable: Survives server restart
  • AutoAck: Automatically mark message as received

🔧 Common Issues & Solutions

❌ "Connection refused" or "BrokerUnreachableException"

Solution:
  • Check CloudAMQP connection string is correct
  • Verify you have internet connection
  • Ensure connection string uses amqps:// (secure)

❌ Messages published but not received

Solution:
  • Start FeeService BEFORE creating enrollments
  • Check FeeService console for "listening for enrollment events"
  • Verify exchange names match in both services

❌ "Invalid object name 'Fees'"

Solution:
  • Run database migration for FeeService
  • dotnet ef database update --project Backend/CMS.FeeService
