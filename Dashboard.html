✨ Key Features & Implementations

🚀 Dapper (Micro-ORM)

10x faster than EF Core for read operations. Attendance Service uses raw SQL queries with Dapper for high-performance database access. No change tracking overhead means ultra-fast bulk operations.

📊 SqlBulkCopy

High-performance bulk insert capability that streams data directly to SQL Server. Can insert 10,000+ records in 2 seconds vs 150 seconds with individual INSERTs. Batch size set to 1000 rows for optimal performance.

📁 CSV/Excel Import

CsvHelper: Flexible column mapping supporting multiple header names (StudentId, studentId, student_id). ClosedXML: Native Excel file processing without Microsoft Office. Both support template downloads for easy data preparation.

📨 Event-Driven Architecture

CloudAMQP (RabbitMQ) enables asynchronous communication between services. When a student enrolls, Enrollment Service publishes event → Fee Service auto-creates $5000 fee record. Services are decoupled and fault-tolerant.

🗄️ Database-per-Service Pattern

Each microservice owns its independent database (clgmansys1-5). Benefits: Security isolation, independent scaling, technology freedom (Dapper for Attendance, EF Core for others), and fault isolation.

🌐 Unified API Gateway

Ocelot-based gateway at port 7000 provides single entry point for all services. Features: Request routing, load balancing, Swagger aggregation with MMLib.SwaggerForOcelot, and CORS configuration.

📨 RabbitMQ Event Flow Details

Student Enrollment Event Flow

Step Action Service Data
1 Student enrolls in course via API Enrollment Service StudentId, CourseId, Semester, Year
2 Create enrollment record in database Enrollment Service EnrollmentId generated
3 Publish event to RabbitMQ Enrollment Service Exchange: "student-enrolled"
4 Message queued and delivered CloudAMQP Fanout pattern (broadcast)
5 Receive and deserialize message Fee Service StudentEnrolledEvent object
6 Auto-create fee record Fee Service Amount: $5000, Status: "Pending"

Message Configuration

Property Value Description
Provider CloudAMQP Cloud-hosted RabbitMQ service
Exchange Name student-enrolled Main event exchange
Exchange Type Fanout Broadcasts to all subscribers
Durable true Survives server restarts
Auto Recovery Enabled (10s interval) Automatic connection recovery
